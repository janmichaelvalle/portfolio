Cafe24 Payment Gateway Integration

  • Category: Product Management
  • Company: Cafe24
  • Website URL: https://global.cafe24.com/

Background Information

As a Product Manager at Cafe24, I led the integration of payment gateways for expansion into new markets like the Philippines, India, and the US. Notable achievements include integrating Maya for the Philippines market and Stripe as a global payment gateway. I collaborated with cross-functional teams to ensure compliance and enhance user experience, driving Cafe24's expansion in the global e-commerce landscape.

Requirements

User Story: As a seller, I want to display payment methods on my online storefront so that customers can easily use them during checkout.
Installation:
  1. Sellers can install the Payment Gateway (PG) app from the Cafe24 app store click the Install button.
Language Selection & User Manual:
  1. Sellers can download the user manual in PDF format from within the app.
  2. Sellers can select their preferred language (EN, KR, VN, JP) from a dropdown menu in the app
PG Account Creation:
  1. Sellers can view details on creating a merchant account and click "Sign-up" to redirect to the provider's website
  2. Tooltips provide additional information on account creation steps.
Connecting PG Company Account:
  1. Sellers can view tooltips for guidance on connecting their PG company account.
  2. They input API keys and click "Link account."
    • If all required API keys are filled, the "Link" button is enabled; otherwise, it's disabled.
  3. After clicking "Link account," they receive system messages based on scenarios:
    • Success message for correct API keys.
    • Failure messages for incorrect keys, duplicate keys, or unexpected errors.
  4. Account linking status:
    • Success:
      • API key input fields are disabled
      • The API key values will be masked showing only the last four digits.
      • The button text and design change to "Unlink account."
    • Failure: API key fields are cleared and page will refresh.
Disconnecting PG Company Account:
  1. Sellers can unlink their PG company account from their Cafe24 store. By clicking the "Unlink account" button, an alert appears.
  2. When the seller interacts with the alert
    • If the seller clicks "OK", the page will refresh, removing API credentials and disabling the payment method in all shops.
    • If the seller clicks "Cancel," the merchant returns to the previous page
Enabling/Disabling Shops:
  1. Sellers can manage the availability of payment methods for their shops by toggling the buttons in the "Display payment method" column.
    • However, only shops with currencies supported by the payment gateway can be enabled.
    • In case of any errors during toggling, an alert message will inform the user, and the toggle button will revert to its previous state.
Payment settings configuration:
  1. By clicking "Payment Settings," sellers are redirected to the Cafe24 admin panel, where they can further configure the payment methods.
User Story#2: As a buyer, I should be able to use the PG company's payment option upon checkout.
Checkout Process
  1. Buyers can select the PG company's payment method during checkout.
  2. Buyers are redirected to the PG company's payment page to complete the transaction.
  3. If the payment is successful, the order is placed.
